# Codebase Structure

**Analysis Date:** 2026-01-14

## Directory Layout

```
get-shit-done-gemini/
├── .claude/            # Claude-specific implementation
├── .gemini/            # Gemini-specific implementation
├── bin/                # Installation scripts
│   └── install.js      # Main installer
├── commands/           # Command source files
│   └── gsd/            # GSD command definitions
├── get-shit-done/      # Core system resources
│   ├── references/     # Methodology and philosophy
│   ├── templates/      # State and document templates
│   └── workflows/      # Procedural logic
├── package.json        # Project manifest
└── GSD-STYLE.md        # System style guide
```

## Directory Purposes

**commands/gsd/:**
- Purpose: Source of truth for GSD command prompts
- Contains: `.md` files defining command objectives and processes

**get-shit-done/workflows/:**
- Purpose: Encapsulated logic for multi-step tasks
- Contains: Detailed execution steps for commands like `map-codebase` or `execute-phase`

**get-shit-done/templates/:**
- Purpose: Blueprint files for project state
- Contains: `project.md`, `roadmap.md`, `config.json`, and `codebase/` sub-templates

**get-shit-done/references/:**
- Purpose: Guiding principles and standards
- Contains: TDD guides, questioning frameworks, and git integration rules

**.gemini/ / .claude/:**
- Purpose: Target-specific implementations
- Contains: Mirrored structure for different AI agents

## Key File Locations

**Entry Points:**
- `bin/install.js`: CLI entry point for installation
- `/gsd:help`: Interactive entry point for discovery

**Configuration:**
- `package.json`: Engine and bin configuration
- `get-shit-done/templates/config.json`: Project-level configuration template

**Core Logic:**
- `get-shit-done/workflows/`: All complex logic resides here

## Naming Conventions

**Files:**
- `kebab-case.md`: Commands and workflows
- `kebab-case.js`: JavaScript sources
- `UPPERCASE.md`: Project-level documentation (README, LICENSE, GSD-STYLE)

**Directories:**
- `kebab-case`: All directories

## Where to Add New Code

**New Command:**
- Definition: `commands/gsd/{name}.md`
- Gemini port: `.gemini/commands/gsd/{name}.toml`

**New Workflow:**
- Implementation: `get-shit-done/workflows/{name}.md`

**New Template:**
- Implementation: `get-shit-done/templates/{name}.md`

## Special Directories

**.planning/:**
- Purpose: (Target project only) Contains the persistent state of the GSD system
- Source: Generated by `/gsd:new-project`
- Committed: Yes
